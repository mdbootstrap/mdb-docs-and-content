使用

模态插件通过数据属性或JavaScript按需切换隐藏的内容。它还增加了 .modal-open 到<body>覆盖默认滚动行为并生成一个 .modal-背景提供一个单击区域，以在模态外部单击时消除显示的模态。

数据属性

无需编写JavaScript即可激活模式。组 data-toggle =“ modal” 在控制器元素（如按钮）上，以及 data-target =“＃foo” 要么 href =“＃foo” 以特定的模式进行切换。

通过JavaScript

用ID调用模式 myModal 用一行JavaScript：

$（&#39;＃myModal&#39;）。modal（options）

选项

可以通过数据属性或JavaScript传递选项。对于数据属性，将选项名称附加到数据- ，如 data-backdrop =“” 。

名称 类型 默认 描述
背景 布尔值或字符串 &#39;静态的&#39; 如果为 true，则幻灯片将没有固定位置 包括模式背景元素。或者，指定静态的对于无法关闭点击模式的背景。
键盘 布尔值 如果为 true，则幻灯片将没有固定位置 按下退出键时关闭模式
焦点 布尔值 如果为 true，则幻灯片将没有固定位置 初始化时将重点放在模态上。
显示 布尔值 如果为 true，则幻灯片将没有固定位置 初始化时显示模态。

异步方法和转换

所有API方法都是异步然后开始过渡 。转换开始后，他们会立即返回给呼叫者，但在结束之前 。此外，在过渡组件将被忽略 。

如果模态的高度在打开时发生更改，则手动重新调整模态的位置（即，如果出现滚动条）。

$（&#39;＃myModal&#39;）。modal（&#39;handleUpdate&#39;）

.modal（&#39;dispose&#39;）

销毁元素的模态。

活动

Bootstrap的模态类公开了一些事件，可以挂接到模态功能上。所有模态事件都会向模态本身触发（即<div class="modal"> ）。

要查看模态事件的更多示例，请阅读模态事件文档

事件类型 描述
show.bs.modal 当节目实例方法被调用。如果是由点击引起的，则clicked元素可用作 relatedTarget 事件的属性。
显示.bs.modal 当模式对用户可见时将触发此事件（将等待CSS转换完成）。如果是由点击引起的，则clicked元素可用作 relatedTarget 事件的属性。
hide.bs.modal 当隐藏实例方法已被调用。
隐藏的.bs。模态的 在向用户隐藏模式后，将触发此事件（将等待CSS转换完成）。
