自举布局

我们在这里介绍用于布局Bootstrap项目的一些组件和选项，包括包装容器，功能强大的网格系统，灵活的媒体对象和响应实用程序类。

容器

容器是Bootstrap中最基本的布局元素，并且在使用我们的默认网格系统时是必需的。从响应式，固定宽度的容器（表示每个断点处的最大宽度发生变化）或流体宽度（表示其始终为100％宽度）中进行选择。

尽管可以嵌套容器，但是大多数布局不需要嵌套容器。

集装箱液体

采用 .container-fluid 对于全角容器，它跨越视口的整个宽度。

响应临界点：

由于Bootstrap首先开发为移动设备，因此我们使用了少数媒体查询来为布局和界面创建合理的断点。这些断点主要基于最小视口宽度，并允许我们随着视口的变化按比例放大元素。

Bootstrap主要在源Sass文件中将以下媒体查询范围（或断点）用于布局，网格系统和组件。

//额外的小型设备（纵向电话，小于576px）//没有媒体查询，因为这是Bootstrap中的默认设置//小型设备（横向电话，576px及以上）@media（最小宽度：576px）{... } //中型设备（平板电脑，768像素及以上）@media（最小宽度：768像素）{... } //大型设备（台式机，992px及以上）@media（最小宽度：992px）{... } //大型设备（大型台式机，1200px及以上）@media（最小宽度：1200px）{... }

由于我们用Sass编写了源CSS，因此所有媒体查询都可以通过Sass mixins获得：

include media-breakpoint-up（xs）{...} @include media-breakpoint-up（sm）{...} @include media-breakpoint-up（md）{...} @include media-breakpoint-up up（lg）{...} @include media-breakpoint-up（xl）{... } //示例用法：@include media-breakpoint-up（sm）{.some-class {display：block; }}

我们有时会使用在另一个方向（给定的屏幕尺寸或更小）上起作用的媒体查询：

//额外的小型设备（纵向电话，小于576px）@media（最大宽度：575.98px）{... } //小型设备（横向电话，小于768px）@media（最大宽度：767.98px）{... } //中型设备（平板电脑，小于992px）@media（最大宽度：991px）。98） } //大型设备（台式机，小于1200px）@media（最大宽度：1199.98px）{... } //大型设备（大型台式机）//没有媒体查询，因为超大断点的宽度没有上限

请注意，由于浏览器当前不支持范围上下文查询 ，我们致力于解决以下限制最小和最大前缀通过使用精度更高的值进行这些比较，可以创建具有小数宽度的视口（例如，在某些条件下会出现在高dpi的设备上）。

同样，这些媒体查询也可以通过Sass mixins使用：

include media-breakpoint-down（xs）{...} @include media-breakpoint-down（sm）{...} @include media-breakpoint-down（md）{...} @include media-breakpoint-down down（lg）{... }

也有媒体查询和混合，用于使用最小和最大断点宽度来定位单个屏幕尺寸段。

//额外的小型设备（纵向电话，小于576px）@media（最大宽度：575.98px）{... } //小型设备（横向电话，576像素及以上）@media（最小宽度：576像素）和（最大宽度：767.98像素）{... } //中型设备（平板电脑，768像素及以上）@media（最小宽度：768px）和（最大宽度：991.98px）{... } //大型设备（台式机，992px及以上）@media（最小宽度：992px）和（最大宽度：1199.98px）{... } //大型设备（大型台式机，1200px及以上）@media（最小宽度：1200px）{... }

这些媒体查询也可以通过Sass mixins使用：

仅包括媒体断点（xs）{...} @包括仅媒体断点（sm）{...} @包括仅媒体断点（md）{...} @包括媒体断点- only（lg）{...} @include media-breakpoint-only（xl）{... }

同样，媒体查询可能跨越多个断点宽度：

//示例//应用样式，从中型设备开始到超大型设备@media（最小宽度：768px）和（最大宽度：1199.98px）{... }

用于定位相同屏幕尺寸范围的Sass mixin为：

包括media-breakpoint-between（md，xl）{... }

Z-指数

一些Bootstrap组件利用索引 ，这是CSS属性，它通过提供第三个轴来安排内容来帮助控制布局。我们在Bootstrap中使用默认的z-index比例尺，该比例尺旨在适当地分层导航，工具提示和弹出窗口，模态等。

这些较高的值从任意数字开始：足够高且具体到足以避免冲突的程度。我们需要在我们的分层组件（工具提示，弹出窗口，导航栏，下拉菜单，模式）中使用一组标准的工具，以便我们在行为上可以合理地保持一致。没有理由我们不能使用 100 +或 500 +。

我们不鼓励定制这些值。如果您要更改一项，则可能需要全部更改。

$ zindex-sticky：1020！默认; $ zindex-fixed：1030！默认; $ zindex-modal-backdrop：1040！默认; $ zindex-modal：1050！默认; $ zindex-popover：1060！默认; $ zindex-tooltip：1070！默认

为了处理组件内的重叠边框（例如输入组中的按钮和输入），我们使用低个位数索引的值 1个 ， 2 和 3 默认，悬停和活动状态。在悬停/焦点/活动上，我们将特定元素以更高的索引显示他们的兄弟元素边界的价值。
