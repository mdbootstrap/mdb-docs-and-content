旋转木马

幻灯片组件，用于循环显示元素（图像或文本幻灯片），例如轮播。

工作原理

该轮播是一个幻灯片演示，用于循环显示一系列内容，这些内容使用CSS 3D转换和一些JavaScript构建。它适用于一系列图像，文本或自定义标记。它还包括对上一个/下一个控件和指示器的支持。

在浏览器中页面可见性API 如果支持，则当用户看不到网页时，轮播会避免滑动（例如，当浏览器选项卡处于非活动状态，浏览器窗口最小化等）时。

请注意，不支持嵌套轮播，轮播通常不符合辅助功能标准。

范例

轮播不会自动标准化幻灯片尺寸。因此，您可能需要使用其他实用程序或自定义样式来适当调整内容的大小。虽然轮播支持上一个/下一个控件和指示器，但并没有明确要求它们。根据需要添加和自定义。

使用

数据属性

使用数据属性可以轻松控制转盘的位置。数据幻灯片接受关键字上一个要么下一个 ，这会相对于当前位置更改幻灯片位置。或者，使用数据滑入将原始幻灯片索引传递到轮播 data-slide-to =“ 2” ，从开始将幻灯片位置移动到特定索引 0 。

的 data-ride =“ carousel” 属性用于将轮播标记为从页面加载开始动画。它不能与同一轮播的（冗余和不必要的）显式JavaScript初始化结合使用。

通过JavaScript

使用以下代码手动调用轮播。

$（&#39;。carousel&#39;）。carousel（）

选项

可以通过数据属性或JavaScript传递选项。对于数据属性，将选项名称附加到数据- ，如 data-interval =“” 。

名称 类型 默认 描述
间隔 数字 5000 自动循环幻灯片之间的延迟的时间量。如果为假，则轮播不会自动循环。
暫停 字符串布尔值 悬停

如果设置为“ hover”，则暂停在mouseenter上的轮播循环，并在mouseleave上恢复轮播的循环。如果设置为false，则将鼠标悬停在转盘上不会暂停它。

在启用触摸的设备上，当设置为“悬停”时，骑行将在触摸端暂停（一旦用户完成与轮播的互动）两个间隔，然后自动恢复。请注意，这是上述鼠标行为的补充。
包装 布尔值 如果为 true，则幻灯片将没有固定位置 轮播是否应该连续循环或是否有硬停止。
游乐设施 字符串 false 用户手动循环第一项后，自动播放轮播。如果为“轮播”，则在加载时自动播放轮播。
键盘 布尔值 如果为 true，则幻灯片将没有固定位置 轮播是否应该对键盘事件做出反应。
摸 布尔值 如果为 true，则幻灯片将没有固定位置 轮播是否应支持触摸屏设备上的向左/向右滑动交互。

方法

异步方法和转换

所有API方法都是异步然后开始过渡 。转换开始后，他们会立即返回给呼叫者，但在结束之前 。此外，在过渡组件将被忽略 。

轮播选项

使用可选选项初始化轮播宾语并开始循环浏览各个项目。

$（&#39;。carousel&#39;）。carousel（{interval：2000}）
.carousel（&#39;cycle&#39;）

从左到右循环浏览轮播项目。

轮播暂停

阻止轮播在各个项目之间循环。

.carousel（number）

将轮播循环到特定帧（从0开始，类似于数组）。

.carousel（&#39;prev&#39;）

循环到上一个项目。

.carousel（&#39;next&#39;）

循环到下一个项目。

.carousel（&#39;dispose&#39;）

销毁元素的轮播。

活动

Bootstrap的轮播类展示了两个挂钩到轮播功能的事件。这两个事件具有以下附加属性：

  1. 方向 ：轮播的滑动方向（任一 “剩下” 要么 “对” ）。
  2. relatedTarget ：作为活动项目滑入到位的DOM元素。
  3. 从 ：当前项目的索引。
  4. 至 ：下一项的索引。

所有轮播事件都会在轮播本身上触发（即，在<div class="carousel"> ）。

事件类型 描述
slide.bs.carousel 当滑动实例方法被调用。
滑动转盘 当轮播已完成其幻灯片过渡时，会触发此事件。
