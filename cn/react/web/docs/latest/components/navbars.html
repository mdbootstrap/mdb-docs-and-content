工作原理

在开始使用导航栏之前，您需要了解以下内容：

  • 导航栏会带来许多道具，包括颜色和扩大用于响应性崩溃。

  • 导航栏及其内容默认情况下是流畅的。采用可选容器限制其水平宽度。

  • 导航栏及其内容是使用flexbox构建的，可通过实用程序类提供简单的对齐选项。

  • 导航栏默认情况下是响应式的，但是您可以轻松地对其进行修改以进行更改。响应行为取决于我们坍方零件。

  • 为了确保可访问性，我们的导航栏使用导航 HTML标签是默认设置，但如果设置了标签支撑其他东西，他们还提供角色=“导航” 属性内。这样，可以将其明确标识为辅助技术用户的界标区域。

请继续阅读以获取示例和支持的子组件列表。

从MBD React版本开始 1.5 ，导航与React-Router兼容。因此，它需要导入 {BrowserRouter} 组件来自反应路由器域在文件的开头，并包装（渲染）的内容返回（） 中的陈述<BrowserRouter>零件。的<NavLink>的至现在需要属性。

支持内容

导航栏附带了对其他一些子组件的支持。根据需要从以下选项中进行选择：

  • <NavbarBrand>您公司，产品或项目的名称。

  • <NavbarNav>全高和轻量级导航（包括对下拉菜单的支持）。

  • <NavbarToggler>与我们一起使用坍方组件及其他导航切换行为。

  • <Collapse>用于通过父断点分组和隐藏导航栏内容。

对于永不崩溃的导航栏，请将其传递给扩大支柱。要使用宽度断点进行折叠，请传递扩大用值之一支持- 短信 ， md ， lg 要么 l 。要使导航栏始终折叠，只需不添加任何导航栏扩大相关道具
